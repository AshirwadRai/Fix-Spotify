# Downloads & Offline

Downloads are queued and tagged.

## Tagged with what you actually saw

**A download embeds exactly what the player showed you** — same artist, same album, same artwork.

That sounds obvious until you've used a downloader that doesn't do it. Files land in your music folder already correct, and play properly in any other music player, phone, or car stereo that reads tags.

## Platform differences

| | What happens |
| --- | --- |
| **Windows** | Audio is converted with the bundled FFmpeg. You never install FFmpeg yourself. |
| **Android** | The original container is kept as-is — **no transcode, no quality loss**, and it finishes faster. |

## Where files go

**Settings → Storage** sets the download folder, and there's a reset-to-default if you move it somewhere awkward.

## Playing offline

Downloaded songs play with **no connection at all**. They appear in your library alongside everything else.

## Removing a download

Delete it from the downloads list and the library updates immediately — no need to leave the screen and come back.

## If a download fails

Occasionally a source will refuse mid-transfer. Retry it; if the track exists on another source the app will generally succeed on the second attempt using a different one.

Persistent failures on one specific track usually mean it only exists on a source you have switched off — try enabling YouTube in **Settings → Sources**.
